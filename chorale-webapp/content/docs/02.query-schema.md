A query is a single JSON object with these fields:

| Field                      | Type                 | Required | Default     | Description                                                 |
|----------------------------|----------------------|----------|-------------|-------------------------------------------------------------|
| `feature`                  | `string`             | yes      | —           | The **driving feature**, see below.                         |
| `pattern`                  | `array` of `objects` | yes      | —           | The sequence to search for. See [Patterns](/docs/patterns). |
| `voices`                   | `string`             | no       | `"all"`     | Which voices to search.                                     |
| `limit`                    | `number`             | no       | *unlimited* | Stop after this many results, across the whole corpus.      |
| `mintStartAtPreviousToken` | `boolean`            | no       | `false`     | See [Options](/docs/options#mintstartatprevioustoken).      |
| `fbCompareExactChord`      | `boolean`            | no       | `false`     | See [Options](/docs/options#fbcompareexactchord).           |
| `kernIgnoreOctave`         | `boolean`            | no       | `false`     | See [Options](/docs/options#kernignoreoctave).              |



## The driving feature

`feature` picks which analysis spine the search actually **walks through**. The
engine collects every onset (every non-empty note/interval/harmony event) in
that spine, for the voices you asked for, and then slides your `pattern` — a
window of one or more positions — over consecutive onsets, checking each
position against one onset.

Only the five spines listed on the [Features](/docs/features) page can be a
driving feature: `kern`, `deg`, `mint`, `fb`, `metweight`. A pattern position
can *additionally* check `duration` and `fermata` and any other feature at each
onset (see [Checking several features at
once](/docs/patterns#checking-several-features-at-once)) but `duration` and
`fermata` cannot be the driving feature themselves, because they aren't
independent spines: they're read off the underlying `**kern` token at whatever
line the driving feature happens to land on. Setting `feature` to `duration` or
`fermata` (or any name that isn't a real spine) always returns zero results.

Two consequences of "the driving feature decides which onsets exist" that come
up a lot in practice:

- The pattern's length is a length in **onsets of that feature**, not in beats
  or notes of some other voice. A 3-position `deg` pattern spans 3 consecutive
  scale-degree changes; the same 3-position pattern driven by `mint` spans 3
  consecutive melodic intervals.
- Switching the driving feature changes *where* matches can start, even for an
  otherwise identical pattern, because it changes what counts as "the next
  onset".



## `voices`

A case-insensitive selector for which voices to search. Voices are numbered 1‑4
from the bottom up: `1` = Bass, `2` = Tenor, `3` = Alto, `4` = Soprano.

| Selector                                   | Meaning                                      |
|--------------------------------------------|----------------------------------------------|
| `"all"` or `""`                            | every voice (default)                        |
| `"1"` … `"4"`                              | a single voice by number                     |
| `"bass"`, `"tenor"`, `"alto"`, `"soprano"` | a single voice by name                       |
| `"b"`, `"t"`, `"a"`, `"s"`                 | a single voice by initial letter             |
| a run of digits, e.g. `"124"`              | several voices at once: Bass, Tenor, Soprano |
| comma-separated, e.g. `"bass,3,s"`         | mix names/digits/letters freely              |

Each matching voice is searched independently and reported separately in the
results (see the `voice`/`voiceLabel` fields returned per result).



## `limit`

Caps the total number of results across the **entire corpus**, not per chorale.
Once the cap is hit, the search stops scanning further chorales altogether.
Useful while iterating on a query, since a typo that matches almost everything
won't force you to wait for all 370 files.
