title: mint
description: Searching by melodic interval.

`**mint` is Humdrum's
[`mint`-tool](https://www.humdrum.org/tool/mint/index.html) for melodic-interval
analysis, run against each voice: each token is the interval from the *previous*
note in that same voice to the current one.



## Token format

A `**mint` token is a signed, qualified diatonic interval:

- an optional leading **`+`** or **`-`** for direction (up or down;
  ties/repeated notes have no sign),
- a **quality** letter `M` (major), `m` (minor), `P` (perfect), `A`
  (augmented), `d` (diminished), and doubled forms for wider deviations (`AA`,
  `dd`, ...),
- a **diatonic number** `2` for a 2nd, `3` for a 3rd, and so on.

So `+M2` is a major 2nd up, `-m3` a minor 3rd down, `P1` a unison (repeated
note).

The very first note of a voice has no previous note to measure from, so instead
of an interval it gets a literal, bracketed pitch name, e.g. `[gg]`. This is
still a real, non-empty onset, it counts as a `mint` onset just like any
interval token, but it won't match a normal interval pattern value; only `"*"`
or the literal bracketed string itself will match it.



## Quality- and sign-optional matching

Pattern values for `mint` don't have to be a complete token. Any part you leave
out is treated as "matches any":

| Pattern value | Matches                                                                |
|---------------|------------------------------------------------------------------------|
| `"M2"`        | a major 2nd, **either direction** (`+M2` or `-M2`)                     |
| `"+2"`        | **any** 2nd going up, regardless of quality (`+M2`, `+m2`, `+A2`, ...) |
| `"2"`         | any 2nd at all, any direction, any quality                             |
| `"+M2"`       | only an ascending major 2nd (fully specified, matches literally)       |

This makes it easy to search for melodic *shape* without caring about exact
diatonic quality, or for a specific quality without caring about direction.

```json
{
    "feature": "mint",
    "pattern": [
        { "mint": "2" },
        { "mint": "2" },
        { "mint": "2" }
    ]
}
```

finds three consecutive stepwise motions of any direction and quality. A generic
"walking in seconds" passage, in any voice.



## Mixing specificity across positions


Nothing forces every position in a pattern to use the same level of detail. Each
position is matched independently, so a fully specified interval can be followed
by a direction-only one, then a quality-only one, and so on:

```json
{
    "feature": "mint",
    "pattern": [
        { "mint": "+2" },
        { "mint": "-M3" },
        { "mint": "P1" },
        { "mint": "3" }
    ]
}
```

matches a step up of any quality, then a descending major 3rd specifically, then
a repeated note, then a 3rd of any quality in either direction. Pin down exactly
as much (or as little) as matters for each note in the run.



## `mint` starts one token late

Because a `mint` onset describes the interval *into* a note, the driving onset
for position 0 of your pattern is already the *second* note of whatever you're
matching. The first note (the one the interval starts from) isn't itself an
onset your pattern checked, so by default it's excluded from the reported
`startPosition`/ `startLine`.

If you want the match to start on that first note instead, either:

- set the query option [`mintStartAtPreviousToken`](/docs/options#mintstartatprevioustoken), or
- put an explicit wildcard `{ "mint": "*" }` as position 0 of the pattern.

See [Options](/docs/options#mintstartatprevioustoken) for the full explanation
and an example output of the difference this makes.



## Using it in a pattern

```json
{
    "feature": "mint",
    "pattern": [
        { "mint": "-2" },
        { "mint": "-2" },
        { "mint": "+3" }
    ]
}
```

matches two consecutive downward steps followed by an upward leap of a 3rd, any
quality throughout. See [Patterns](/docs/patterns) for OR-lists, wildcards,
multi-feature positions, and negation, all of which work the same way for `mint`
as for any other feature.
