---
title: metweight
description: Searching by metric weight (beat strength).
---

`**metweight` is Humdrum's metric-weight tool run once per chorale: it
classifies every note/rest attack in a `**kern` voice by how strong its beat
position is within the current meter.



## Token format

| Weight class | Meaning                                                    |
|--------------|------------------------------------------------------------|
| strong       | the beat itself (e.g. beat 1 of the measure)               |
| half-strong  | a secondary strong position (e.g. beat 3 of a 4/4 measure) |
| weak         | a week position (e.g. beat 2 and 4 of a 4/4 measure)       |
| unclassified | any other position                                         |



## Three ways to write a weight class

A pattern value can spell out a weight class as an abbreviation, a full word, or
a numeric rank. All three are equivalent, since `metweight` always normalizes
down to the abbreviation before comparing (the actual `**metweight` data is
written in abbreviated form):

| Weight class | Abbreviation | Full word        | Number |
|--------------|--------------|------------------|--------|
| strong       | `"s"`        | `"strong"`       | `"1"`  |
| half-strong  | `"hs"`       | `"half-strong"`  | `"2"`  |
| weak         | `"w"`        | `"weak"`         | `"3"`  |
| unclassified | `"u"`        | `"unclassified"` | `"4"`  |

```json
{
    "feature": "metweight",
    "pattern": [
        { "metweight": "strong" }
    ]
}
```

matches the same onsets as `{ "metweight": "s" }` or `{ "metweight": "1" }`.



## Using it in a pattern

`metweight` is most useful either as the driving feature (walking beat-to-beat
through the metric structure) or as a secondary constraint alongside
`kern`/`deg`/`mint`/`fb` (checking "does this melodic thing land on a strong
beat"):

```json
{
    "feature": "kern",
    "voices": "bass",
    "pattern": [
        { "fb": ["7", "4", "2", "9"], "metweight": ["s", "hs"] }
    ]
}
```

matches every dissonance that lands on a strong of half-strong beat. See
[Patterns](/docs/patterns) for OR-lists, wildcards, multi-feature positions, and
negation, all of which work the same way for `metweight` as for any other
feature.
