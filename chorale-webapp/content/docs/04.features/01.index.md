Every chorale is analyzed once when it's loaded, producing several parallel
Humdrum spines. Each spine is a **feature** you can use as the [driving
feature](/docs/query-schema#the-driving-feature) or reference from any pattern
position:

| Feature                                 | Description                                                                          | Driving feature |
|-----------------------------------------|--------------------------------------------------------------------------------------|-----------------|
| [`kern`](/docs/features/kern)           | The notes themselves (rhythm + pitch)                                                | yes             |
| [`deg`](/docs/features/deg)             | Scale degree relative to the local key                                               | yes             |
| [`mint`](/docs/features/mint)           | Melodic interval from the previous note                                              | yes             |
| [`fb`](/docs/features/fb)               | Figured-bass harmony above the bass                                                  | yes             |
| [`metweight`](/docs/features/metweight) | Metric weight / beat strength (`"strong"`/`"half-strong"`/`"weak"`/`"unclassified"`) | yes             |
| `duration`                              | Rhythmic value (Humdrum recip, tied notes combined, e.g. `"4"`, `"8"`)               | no              |
| `fermata`                               | Whether the note carries a fermata (`"true"`/`"false"`)                              | no              |

`duration` and `fermata` are read directly off the underlying `**kern` token for
whichever line a driving-feature onset lands on. They aren't independent spines,
so they can only appear as extra constraints inside a pattern position (see
[Checking several features at
once](/docs/patterns#checking-several-features-at-once)), never as `feature`
itself.

Each of the five spine-backed features has its own page with the exact token
format and how to write pattern values against it.
