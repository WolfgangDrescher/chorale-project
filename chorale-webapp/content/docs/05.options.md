Three query-level fields that don't belong to `pattern` itself: two toggles that
change how a match's boundaries or comparisons are computed, and one cap on how
much of the corpus gets searched.



## `mintStartAtPreviousToken`

**Type:** `boolean` · **Default:** `false`

Only affects searches driven by [`mint`](/docs/features/mint). A `**mint` token
describes the interval *into* a note, so the first onset your pattern checks is
already the second note of the phrase you're matching. The note the interval
starts *from* isn't itself checked by position 0, and by default it's left out
of the reported `startPosition`/`startLine`.

Setting `mintStartAtPreviousToken: true` shifts the reported start back by one
onset, so the match begins on the note the first interval was measured from, not
the note it lands on.

```json
{
    "feature": "mint",
    "voices": "soprano",
    "mintStartAtPreviousToken": true,
    "pattern": [
        { "mint": "-2" },
        { "mint": "-2" }
    ]
}
```

Without the option, a match's `startLine` points at the *second* note of the
descending-2nds run; with it, `startLine` points at the *first* note, one note
earlier, so the highlighted range covers all three notes of the run instead of
just the last two:

::code-group

```vue [without the option]

chor029  mint     Soprano  30          32        4               5
chor029  mint     Soprano  32          34        5               6
chor029  mint     Soprano  34          36        6               7
chor029  mint     Soprano  46          48        12              13
chor029  mint     Soprano  48          50        13              14
chor029  mint     Soprano  57          60        19              20
chor029  mint     Soprano  60          61        20              21
chor029  mint     Soprano  73          76        27              28
chor029  mint     Soprano  76          78        28              29
chor029  mint     Soprano  78          80        29              30
chor029  mint     Soprano  87          89        34              35
chor029  mint     Soprano  89          90        35              35+1/2
```

```vue [with mintStartAtPreviousToken]

chor029  mint     Soprano  26          32        3               5
```

**The escape hatch**: the shift is skipped when position `0` of your pattern
explicitly sets `mint` to a wildcard, `{ "mint": "*" }`, so that it doesn't add
one extra step on top of a position you've already accounted for yourself. Any
other position `0` (a specific `mint` value, or no `mint` key at all, e.g. only
checking `duration` or another feature) still gets the shift.



## `fbCompareExactChord`

**Type:** `boolean` · **Default:** `false`

Only affects comparisons against [`fb`](/docs/features/fb). By default, a
multi-figure pattern value like `"2 4"` is a **minimum** requirement. It matches
any chord that contains *at least* those figures, so it also matches an actual
chord of `"2 4 6"`. Setting `fbCompareExactChord: true` requires the actual
chord to have **exactly** as many figures as the pattern value, no more:

```json
{
    "feature": "fb",
    "fbCompareExactChord": true,
    "pattern": [
        { "fb": "m6 m3" }
    ]
}
```

With the default (`false`), this also matches a four-note chord that happens to
contain both a minor 6th and a minor 3rd above the bass, alongside other
figures. With `fbCompareExactChord: true`, it only matches chords that are
*exactly* a minor 6th and minor 3rd, nothing else sounding above the bass.



## `limit`

**Type:** `number` · **Default:** unlimited

Caps the total number of results returned, counted across the **entire corpus**,
not per chorale. As soon as the cap is reached, the search stops scanning
further chorales; files are processed in sorted filename order, so `limit`
always returns the same first N results for a given query, deterministically.

```json
{
    "feature": "deg",
    "limit": 20,
    "pattern": [
        { "deg": "7" },
        { "deg": "1" }
    ]
}
```

`limit` is mainly useful while you're iterating on a query: a pattern that's too
loose (or has a typo that makes it match almost everything) won't force you to
wait for all 370 chorales to be scanned before you notice.
